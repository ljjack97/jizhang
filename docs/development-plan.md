# 开发执行计划

## 开发原则
- **小步推进**：每个 Phase 完成并验证后再进入下一个
- **可运行**：每个 Phase 结束时代码必须可运行、无报错
- **先骨架后血肉**：先搭框架，再加功能，最后做美化

---

## Phase 1：项目脚手架初始化

**目标**：创建一个能跑起来的空项目

- [ ] 使用 Vite 创建 Vue 3 项目
- [ ] 安装核心依赖：vue-router, pinia, vant
- [ ] 配置 Vant 按需引入（unplugin-vue-components）
- [ ] 配置 vite.config.js
- [ ] 创建基础目录结构（router/, stores/, views/, components/, api/, utils/, styles/）
- [ ] 编写全局样式 global.css（主题色变量）
- [ ] 配置 index.html 移动端 viewport
- [ ] 验证：`npm run dev` 能看到空白页面

## Phase 2：基础设施搭建

**目标**：路由、状态管理、数据层就绪

- [ ] 创建路由配置（5 个页面路由）
- [ ] 创建 Pinia Record Store（增删查 + localStorage 持久化）
- [ ] 创建 localStorage 工具函数
- [ ] 创建 API 接口层（mock 实现，CRUD 方法）
- [ ] 创建 App.vue 根布局（router-view + 底部导航）
- [ ] 创建 AppHeader 公共组件
- [ ] 验证：页面之间可以跳转，Store 可以读写数据

## Phase 3：首页 - 支付记录列表

**目标**：首页能看到记录列表

- [ ] 创建 Home.vue 页面
- [ ] 创建 RecordCard.vue 组件
- [ ] 创建 EmptyState.vue 组件
- [ ] 实现记录列表渲染（从 Store 读取）
- [ ] 实现时间倒序排列
- [ ] 实现下拉刷新
- [ ] 补充 mock 初始数据（3-5 条示例记录）
- [ ] 验证：首页展示记录列表，空数据时显示空状态

## Phase 4：添加记录页

**目标**：能手动录入支付记录

- [ ] 创建 Add.vue 页面
- [ ] 表单布局（金额、时间、订单号、平台、备注）
- [ ] 表单校验规则
- [ ] 提交逻辑（写入 Store + localStorage）
- [ ] 提交成功提示 + 跳转首页
- [ ] 验证：添加记录后首页能显示新记录

## Phase 5：记录详情页

**目标**：查看详情和删除

- [ ] 创建 Detail.vue 页面
- [ ] 展示完整记录信息
- [ ] 删除功能（确认弹窗 + 删除 + 返回首页）
- [ ] 从首页卡片点击跳转详情（路由传参）
- [ ] 验证：点击卡片 → 看到详情 → 删除 → 返回首页并刷新

## Phase 6：搜索功能

**目标**：能按关键词和日期搜索

- [ ] 创建 Search.vue 页面
- [ ] 搜索输入框（关键词）
- [ ] 日期范围选择器
- [ ] 搜索结果列表（复用 RecordCard）
- [ ] 首页导航栏添加搜索入口
- [ ] 验证：搜索关键词/日期能正确筛选记录

## Phase 7：设置页 & 爬虫预留

**目标**：预留爬虫配置界面

- [ ] 创建 Settings.vue 页面
- [ ] 自动抓取开关（UI 占位）
- [ ] 平台多选（微信/支付宝/银行卡）
- [ ] 抓取状态显示区
- [ ] 手动触发按钮（UI 占位）
- [ ] 底部导航添加设置入口
- [ ] 验证：设置页可访问，开关可切换

## Phase 8：PWA 配置 & 移动端优化

**目标**：可安装到手机桌面

- [ ] 安装 vite-plugin-pwa
- [ ] 配置 PWA manifest（应用名、图标、主题色）
- [ ] 生成应用图标
- [ ] 配置 Service Worker 缓存策略
- [ ] 移动端全面测试 & 样式微调
- [ ] 验证：手机浏览器打开 → 可安装到桌面 → 离线可访问

---

## 完成标准

每个 Phase 完成后必须满足：
1. `npm run dev` 无报错
2. 该 Phase 所有功能点可用
3. 样式与设计规范一致
4. 更新当日开发日志
