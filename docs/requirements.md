# 开发需求文档

## 项目名称
记账软件（JiZhang）

## 项目目标
开发一款运行在手机上的个人记账应用，支持手动录入和（未来）自动爬取支付信息，以时间排序展示所有支付记录，提供搜索和详情查看功能。

## 目标用户
个人用户，需要追踪和管理日常支付记录。

## 功能需求

### FR-01 支付记录列表（首页）
- 所有记录按支付时间倒序排列（最新在上）
- 每条记录卡片展示：金额、支付时间、支付平台标签、订单编号（截断）
- 支持下拉刷新
- 空数据时显示引导提示
- 点击卡片进入详情页

### FR-02 记录详情
- 展示完整字段：金额、支付时间、订单编号、支付平台、备注
- 提供删除按钮，删除后返回首页并刷新

### FR-03 手动添加记录
- 表单字段：
  - 金额（必填，数字输入，保留两位小数）
  - 支付时间（必填，默认当前时间，日期选择器）
  - 订单编号（可选，文本输入）
  - 支付平台（必填，下拉选择：微信/支付宝/银行卡/其他）
  - 备注（可选，文本输入）
- 提交前表单校验
- 提交成功后跳转首页

### FR-04 搜索功能
- 关键词搜索：匹配订单编号、备注
- 日期范围筛选：选择起止日期
- 搜索结果按时间倒序排列

### FR-05 设置页（爬虫预留）
- 自动抓取开关（UI 占位，功能待后端实现）
- 平台选择（微信/支付宝/银行卡）
- 上次抓取时间显示
- 手动触发抓取按钮（UI 占位）

### FR-06 数据持久化
- 使用 localStorage 存储支付记录
- 刷新页面或关闭浏览器后数据不丢失

### FR-07 PWA 支持
- 可安装到手机桌面
- 支持离线访问（已加载过的页面）
- 桌面图标和应用名称

## 非功能需求

### NFR-01 移动端适配
- 以手机屏幕为主要目标，宽度 375-414px 为最佳体验区间
- 在平板和桌面端也能正常显示（居中窄版布局）

### NFR-02 性能
- 首页列表支持虚拟滚动（记录超过 100 条时）
- 首次加载时间 < 2 秒

### NFR-03 可用性
- 界面简洁直观，无需用户手册即可使用
- 表单错误提示清晰
- 操作有即时反馈（加载状态、成功/失败提示）
